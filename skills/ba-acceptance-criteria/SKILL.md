# BA Acceptance Criteria

Use this skill when requirements need verifiable conditions of satisfaction.

## Workflow
1. Read the requirement or user story carefully.
2. Identify success, failure, and boundary cases.
3. Write criteria in Given/When/Then or rule format.
4. Remove ambiguity and hidden assumptions.
5. Align with definition of done or test intent.
6. Confirm the criteria are measurable and realistic.

## Deliverables
- Acceptance criteria set
- Boundary and negative cases
- Review checklist

- Each criterion has one outcome
- Edge cases are not omitted
