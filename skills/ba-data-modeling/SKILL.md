# BA Data Modeling

Use this skill when data relationships, movement, or definitions need to be clarified.

## Workflow
1. Identify core entities, attributes, and relationships.
2. Map data sources, consumers, and transformations.
3. Document data rules, quality constraints, and ownership.
4. Create ERD and data flow views as needed.
5. Capture interface and API assumptions.
6. Validate terminology with business and technical stakeholders.

## Deliverables
- ERD
- Data flow diagram
- Data dictionary
- Data quality notes

- Relationships are unambiguous
- Data rules match the process and requirements
