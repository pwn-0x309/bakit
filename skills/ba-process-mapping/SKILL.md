# BA Process Mapping

Use this skill when you need to visualize how work flows today and how it should change.

## Workflow
1. Define the process boundary and trigger.
2. Identify actors, handoffs, systems, and decisions.
3. Map the AS-IS process step by step.
4. Mark bottlenecks, waste, rework, and delays.
5. Design the TO-BE process with improvements.
6. Capture metrics, owners, and follow-up actions.

## Deliverables
- AS-IS process map
- TO-BE process map
- Swimlane diagram
- Improvement notes and bottlenecks

- The process has a clear start and end
- Handoffs and decisions are explicit
- Improvements are tied to observed issues
