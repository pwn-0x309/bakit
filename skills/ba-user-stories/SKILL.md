# BA User Stories

Use this skill when the work needs Agile story slicing, backlog shaping, or acceptance criteria.

## Workflow
1. Identify the user role, goal, and business value.
2. Decompose epics into features and stories.
3. Write stories in the standard role-goal-benefit format.
4. Add acceptance criteria in Given/When/Then style.
5. Check stories against INVEST.
6. Confirm story size, dependencies, and testability.

## Deliverables
- Epic map
- Story list with acceptance criteria
- Story mapping notes
- Ready/Done checklist

- Acceptance criteria are observable and testable
- Stories are small enough for planning
